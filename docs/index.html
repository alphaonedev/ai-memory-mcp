How It Works

Memories flow through three tiers, from ephemeral to permanent. TTLs decay over time, but each access extends the expiry. Frequently recalled memories auto-promote upward.

Architecture

A single Rust binary exposing both CLI and HTTP API. SQLite with FTS5 handles storage and full-text search. Garbage collection runs automatically.

Memory Lifecycle

Watch a memory evolve from creation to permanent storage. Each recall extends the TTL and increments the access count. At 5 accesses, mid-tier memories auto-promote to long-term.

Recall Scoring

When you recall memories, a composite score determines which ones surface. Five signals combine to rank results.

The Formula

score =
  (fts_rank * -1)         -- BM25 relevance
  + (priority * 0.5)       -- 1-10 scale
  + (access_count * 0.1)  -- frequency
  + (confidence * 2.0)    -- 0.0-1.0
  + tier_boost             -- long=3 mid=1 short=0

FTS5's BM25 algorithm measures text relevance. Priority and confidence are author-set weights. Access count rewards frequently recalled knowledge. The tier boost ensures long-term memories -- which have proven their value -- always surface first.

Quick Start

Three steps. Under a minute.

Install

Requires Rust 1.75+. Install directly from the repository:

cargo install --git https://github.com/alphaonedev/claude-memory.git

Start the daemon

Launch on 127.0.0.1:9077:

claude-memory serve

Or use the CLI directly (no daemon needed):

claude-memory store -T "Project uses Axum 0.8" \
  -c "HTTP layer is Axum 0.8 with tower-http middleware." \
  --tier long --priority 7

Recall what you know

Ask the memory system about any topic:

# CLI
claude-memory recall "HTTP framework choice"

# HTTP API
curl "http://127.0.0.1:9077/api/v1/recall?context=HTTP+framework"

API Reference

All endpoints prefixed with /api/v1. Default: 127.0.0.1:9077. All responses are JSON.

MethodEndpointDescription
GET /health Health check -- DB accessibility + FTS integrity
POST /memories Create a memory. Body: title, content (required); tier, namespace, tags, priority, confidence, ttl_secs
POST /memories/bulk Bulk create. Body: array of memory objects
GET /memories List memories. Params: namespace, tier, limit, offset, min_priority, since, until, tags
GET /memories/{id} Get a memory by ID (returns memory + links)
PUT /memories/{id} Update a memory. Body: any subset of fields
DELETE /memories/{id} Delete a memory by ID
GET /search FTS search (AND). Params: q (required), namespace, tier, limit, min_priority, since, until, tags
GET /recall Recall (OR + touch). Params: context (required), namespace, limit, tags, since
POST /recall Recall via POST body: context, namespace, limit, tags, since
POST /forget Bulk delete. Body: namespace, pattern, tier (at least one required)
POST /consolidate Merge memories. Body: ids (2+ required), title, summary, namespace, tier
POST /links Link two memories. Body: source_id, target_id, relation
GET /links/{id} Get all links for a memory
GET /namespaces List all namespaces with memory counts
GET /stats Database statistics: counts by tier/namespace, DB size, expiring soon
POST /gc Trigger garbage collection of expired memories
GET /export Export all memories and links as JSON
POST /import Import memories and links from JSON

CLI Reference

Every API operation has a CLI equivalent. Global flags: --db <path> and --json.

serve

Start the HTTP daemon

claude-memory serve --port 9077

store

Store a new memory

claude-memory store -T "Title" -c "Content" \
  --tier mid -p 7 --tags a,b

recall

Fuzzy OR search with touch + auto-promote

claude-memory recall "search context" \
  -n my-app --limit 10

search

Exact AND full-text search

claude-memory search "exact terms" \
  --tier long --limit 20

list

List memories with optional filters

claude-memory list -n my-app -t long

get / delete

Retrieve or delete by ID

claude-memory get <uuid>
claude-memory delete <uuid>

promote

Promote a memory to long-term (permanent)

claude-memory promote <uuid>

forget

Bulk delete by namespace, pattern, or tier

claude-memory forget -n my-app \
  --pattern "deprecated" -t short

link

Link two memories with a relation type

claude-memory link <src-id> <tgt-id> \
  -r supersedes

consolidate

Merge multiple memories into one

claude-memory consolidate "id1,id2,id3" \
  -T "Summary" -s "Combined..."

stats / gc / namespaces

Monitoring and maintenance

claude-memory stats
claude-memory gc
claude-memory namespaces

export / import

Full JSON backup and restore

claude-memory export > backup.json
claude-memory import < backup.json
