## 4. Storage / SAL (§04)

Schema **v53** on both backends (`migrations.rs:554` i64 sqlite-side, `postgres.rs:433`
i32 postgres-side; one logical schema number, divergent on-disk file counters). The

`federation_nonce_cache`. `SqliteStore` `Transaction` is a no-op wrapper.

## 5. Governance / Policy / Audit (§05) — PE adjudication crux

The policy engine **exists** but is **not a universal gate**. It is wired through
**two `OnceLock` hooks only**:

- `storage::GOVERNANCE_PRE_WRITE` (`src/storage/mod.rs:97`) — gates memory writes.
- `wire_check::GOVERNANCE_PRE_ACTION` (`src/governance/wire_check.rs:77`) — gates 4
  egress sinks: hooks-spawn, LLM-HTTP, federation-POST, skill-export.

`AgentAction` (`src/governance/agent_action.rs:99`) variants =
`Bash`/`Custom`/`FilesystemWrite`/`NetworkRequest`/`ProcessSpawn` — **no `Read`
variant**. `Decision` (`:188`) = `Allow`/`Refuse`/`Warn`/`Deny`/`Modify` — **no
`Escalate` variant**. Refusals **are** chain-logged + signed: `emit_check_event`
(`:730`) → `append_signed_event` (`src/signed_events.rs:565`).

**Audit-chain verifiers present:**
- File-based `audit.rs:731` — operator-callable via `ai-memory audit verify`.
- `signed_events::verify_chain` (`src/signed_events.rs:396` and `:483`) — library/test
  only, **no CLI verb** (drift D3).
- Reflection-link `verify-reflection-chain` — operator-callable.

## 7. Cognition / Lifecycle (§07)

Hybrid recall = FTS5 keyword + HNSW semantic, blended adaptively (semantic weight
0.50→0.15 by content length), cross-encoder reranker `ms-marco-MiniLM-L-6-v2`. Dual
embedders: MiniLM-L6-v2 (384d) + nomic-embed-text-v1.5 (768d). **HNSW is in-memory
only** — persistence/sharding (G2/G3/G4) deferred to v0.9. Recursive learning
(`memory_reflect`) with depth cap 3. #1389 L2 `recover_from_transcript`.
`InferenceBackend` is a v0.8 seam (`GpuBackend` stubbed, NOT on the v0.7.0 hot-path).
